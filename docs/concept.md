# Концепція Semantic Graphics

## Ідея
Замість малювання пікселів художник компонує зображення з *семантичних атомів*:
- Об’єкти (рука, прапор, долоня, древко)
- Дії (тримати, стискати, торкатися)
- Властивості (розмір, положення, сторона)
- Ролі (агент, інструмент)

## Приклад
Користувач бачить:
- Об’єкт "рука"
- Об’єкт "прапор"

Він перетягує мишкою → з’являється пропозиція взаємодії:
РУКА —[тримає]→ ПРАПОР

Алгоритм визначає:
1. Агент = рука  
2. Пацієнт = прапор  
3. Тип взаємодії = тримання  
4. Потрібні підоб’єкти = долоня, пальці  
5. Потрібний контактний контур = древко прапора  

Система автоматично генерує:
- долоню,
- хват пальців,
- деформацію для “стискання”.

## Цілі
- Зображення = граф знань
- Художник = композитор сенсів
- Об’єкти → ролі  
- Взаємодії → алгоритми
